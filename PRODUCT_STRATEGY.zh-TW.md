# 產品策略

FolderSorter 不應該靠「比開發者工具有更多自動化功能」取勝，而是要讓一般
Mac 使用者敢拿它整理真實資料夾。

## 產品承諾

FolderSorter 是一個 Mac 原生、預覽優先、可復原、本機處理的檔案整理工具。
它要讓使用者知道每個檔案會去哪、為什麼會去那裡，以及整理錯時如何復原。

```text
FolderSorter 要做到的不是最強，而是最敢按。
```

## 目標使用者

第一個客群是 Downloads 爆滿、桌面混亂、截圖一堆、PDF、ZIP、DMG、影片和
圖片散落各處的一般 Mac 使用者。

| 問題 | 產品回答 |
| --- | --- |
| 這是什麼？ | 整理 Downloads、桌面、截圖、文件、媒體、安裝檔的工具。 |
| 會不會馬上亂動檔案？ | 不會，會先預覽。 |
| 整理錯怎麼辦？ | 可以復原最近一次整理。 |
| 檔案會不會上傳？ | 不會，本機處理。 |
| 需要寫規則嗎？ | 不需要，規則應該先視覺化，進階使用者再匯入 / 匯出。 |

## 早期產品不做什麼

FolderSorter 早期不應該把自己定位成：

- Hazel 替代品；
- `organize` 替代品；
- 完整自動化平台；
- 只有開發者會用的 CLI 工具；
- AI 檔案助理。

第一個切入點應該是：想整理混亂 Mac 資料夾、但怕檔案消失的一般使用者。

## 信任原則

### 預覽必須看得懂

Apply 之前，使用者應該能看到：

| 項目 | 要求 |
| --- | --- |
| 原始位置 | 檔案現在在哪裡。 |
| 目標位置 | 檔案會去哪裡。 |
| 動作 | 複製、移動、跳過、重新命名。 |
| 命中原因 | 為什麼這個檔案會分到這個分類。 |
| 衝突處理 | 目標已有同名檔案時會怎麼辦。 |
| 風險提示 | 大量操作、Move 模式、衝突、未知類型、權限問題。 |

產品不應只顯示 `35 files will be sorted`。它應該用人話說明分類數量、
未分類檔案，以及是否會移動或刪除任何檔案。

### Apply 前需要最後確認

最後確認應該依風險調整語氣：

| 情境 | 應有提示 |
| --- | --- |
| Copy 模式 | 低風險確認。 |
| Move 模式 | 明確提醒檔案會離開原位置。 |
| 超過 100 個檔案 | 大量操作提示。 |
| 有同名衝突 | 顯示目前衝突策略。 |
| 有未分類檔案 | 說明不會被亂丟。 |

### Undo 是核心賣點

Undo 應該可靠到可以直接對外溝通：

| 標準 | 目標 |
| --- | --- |
| 可復原範圍 | 至少復原最近一次 Apply。 |
| 歷史可見性 | 使用者看得到最近一次整理改了什麼。 |
| Undo 預覽 | 復原前知道哪些檔案會被搬回。 |
| 失敗處理 | 復原失敗時列出具體原因。 |
| 交易紀錄 | 每次 Apply 都有本機持久紀錄。 |

### 不主打刪除

早期產品應該支援 copy、move、skip、同名自動改名；不應把 delete、trash、
auto clean、auto purge 當主功能。信任來自整理檔案，不是替使用者亂丟檔案。

## 核心體驗

預覽畫面就是產品本體。它應該像一份整理提案，而不是技術操作清單。

| 層級 | 目的 |
| --- | --- |
| 總覽 | 依圖片、文件、壓縮檔、安裝檔、影片、未分類顯示數量。 |
| Before / After | 並排顯示原始位置與預計目標位置。 |
| 原因 | 解釋每個檔案命中的規則。 |
| 例外與風險 | 顯示未分類、衝突、權限失敗、檔案鎖定、資料夾無法建立。 |

使用者看完預覽後，應該能說：我知道它要做什麼，而且我知道它不會亂來。

## 規則策略

規則要簡單，但不能幼稚。

| 等級 | 能力 | 產品期待 |
| --- | --- | --- |
| Level 1 | 副檔名、檔名包含、檔案大類、目標資料夾 | 穩定整理常見 Mac 混亂檔案。 |
| Level 2 | 建立日期、修改日期、檔案大小、來源資料夾、截圖偵測、安裝檔偵測 | 使用者用 GUI 建規則，不需要 JSON 或 YAML。 |
| Level 3 | EXIF 日期、PDF 文字、重複檔偵測、智慧建議、規則包 | 信任體驗成熟後再做的能力。 |

進階使用者仍可匯入 / 匯出 JSON 規則，但預設產品體驗不應要求一般使用者寫
JSON 或 YAML。

## 第一次使用體驗

第一次使用應該在 1 分鐘內讓使用者從混亂資料夾走到預覽。

| 模式 | 用途 |
| --- | --- |
| 整理 Downloads | 最常見整理情境。 |
| 整理 Desktop | 常見視覺混亂情境。 |
| 自訂資料夾 | 進階或特定資料夾整理。 |

| 安全等級 | 意義 |
| --- | --- |
| 最安全 | 只複製，不動原檔。 |
| 標準 | 移動檔案，但可 Undo。 |
| 進階 | 自訂規則與衝突處理。 |

## 品質標準

FolderSorter 會碰使用者本機檔案，品質標準必須比一般工具更高。

- 絕不非預期覆蓋檔案；
- 絕不在沒有交易紀錄時移動檔案；
- Apply 中斷時不能留下狀態不明；
- 絕不遺失最近一次 Undo 紀錄；
- 同名檔案必須透過 skip、rename 或明確 replace 處理；
- 權限與檔案存取失敗要用人話說明；
- 完成後顯示成功、跳過、失敗、新增資料夾、是否可復原的報告。

## 成熟度模型

| 等級 | 意義 |
| --- | --- |
| Level 1：可用工具 | 使用者可以安全整理 Downloads：選資料夾、預覽、copy、move、undo、預設分類、本機處理、同名不覆蓋。 |
| Level 2：可信任工具 | 使用者敢拿它整理真實資料夾：清楚預覽、Apply 報告、Undo 報告、人話錯誤、穩定交易紀錄、可安裝 Mac app、簽章、GUI 規則編輯。 |
| Level 3：好用產品 | 使用者願意定期使用：規則模板、日期與大小條件、截圖偵測、大檔案整理、設定保存、完整雙語介面、接近原生 Mac 的視覺體驗。 |
| Level 4：可成長平台 | 未來的智慧規則建議、重複檔偵測、本機 PDF 分析、EXIF 分類、可分享規則包、plugin、可控的排程整理。 |

Level 4 不是現在的重點。

## 與 `organize` 的差異

`organize` 是給願意寫規則的人。FolderSorter 是給只想把資料夾整理乾淨、而且
想先看懂結果的人。

| 面向 | FolderSorter 目標 |
| --- | --- |
| 上手難度 | 第一次整理不需要讀文件。 |
| 安全感 | 比 CLI 自動化更讓人放心。 |
| 視覺預覽 | 比文字模擬更直覺。 |
| Undo | 成為核心賣點。 |
| 一般人友善 | 明顯勝過 YAML-first 工具。 |
| macOS 體驗 | 像真正的 Mac app。 |
| 隱私 | 清楚本機處理，不上傳、不追蹤。 |

## 成功指標

不要用功能數量判斷成功，要用信任判斷：

| 面向 | 指標 |
| --- | --- |
| 第一次使用 | 使用者能在 1 分鐘內看到預覽。 |
| Apply 信心 | 使用者按下前看得懂整理計畫。 |
| Undo 清楚度 | 使用者知道如何復原最近一次整理。 |
| 錯誤安心感 | 錯誤訊息不讓使用者以為檔案壞了。 |
| 重複使用 | 使用者願意每週或每月再整理一次。 |
| 衝突安全 | 絕不非預期覆蓋檔案。 |
| 本機信任 | 使用者清楚知道檔案不會上傳。 |
