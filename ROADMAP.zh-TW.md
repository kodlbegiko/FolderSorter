# Roadmap

## 最高優先級下一步

這幾項會優先做，因為 FolderSorter 應該先贏得一般 Mac 使用者，而不是變成
另一個只適合進階使用者的自動化工具。

### 可下載版本

目前最大的採用門檻是沒有 `.dmg` 或 `.zip` 下載版。多數一般使用者不會執行
`swift build`，所以下一個公開里程碑應該先透過 GitHub Releases 提供可直接
下載的 app bundle。

交付項目：

- 建立可重現的 `FolderSorter.app` release script。
- 先發布 zipped app build，再補 `.dmg` packaging。
- 文件化未簽章 build 在 macOS 第一次開啟時會遇到的安全提示。
- release 流程穩定後，再往簽章與 notarization 前進。

### 強化視覺化 GUI 規則

規則系統應該變得更強，但不能逼使用者寫 YAML 或 script。GUI 規則仍然要
保持可視覺化、可理解。

預計加入的規則條件：

- 檔案大小
- 建立日期
- 修改日期
- 是否為截圖
- 是否為重複檔
- 圖片、影片、文件、壓縮檔、安裝檔、音訊、程式碼等大類型

### 整理前後對照

預覽不應該只有一串操作清單。整理前後對照能讓使用者在碰到檔案前，就清楚
看懂整理結果。

預計體驗：

- 左側：原本混亂的資料夾結構。
- 右側：預計整理後的輸出結構。
- 標示移動、複製、略過、重複檔、同名衝突項目。
- 最後套用步驟仍然要明確，且可復原。

## V1

- 移動或複製檔案前先產生安全預覽
- 一鍵復原 ledger
- GUI 與 CLI
- JSON 規則匯入/匯出
- 同名衝突策略：自動改名、略過、取代
- 一般 Mac 混亂檔案的預設規則

## V1.1

- 簽章 release builds
- 使用 hash 做更好的重複檔偵測
- 更多規則包：學生、創作者、辦公工作者、開發者
- 可選的排程掃描，但仍維持 preview-first 確認

## V2

- 依媒體長度、圖片尺寸、EXIF 日期分類
- 可選 OCR 與本機文字擷取
- 從近期手動整理紀錄建議規則
- 進階本機處理 plugin API
